# 압축·최적화 적용 근거 (화질 유지)

## 1. 이미지 (JPEG/PNG)

| 근거 | 출처·기준 |
|------|------------|
| **JPEG 품질 82~85** | Google PageSpeed, Squoosh 권장. 웹에서 시각적 차이 거의 없음(visually lossless). |
| **PNG 압축** | sharp 기본 압축(pngCompressionLevel) 또는 무손실 리필터로 용량만 감소. |
| **동일 해상도 유지** | 리사이즈 없이 인코딩만 변경 → 화질 이상 없음. |

→ **적용: JPEG quality 85, PNG 압축. 원본 덮어쓰기(백업 없음).**

## 2. 영상 (MP4)

| 근거 | 출처·기준 |
|------|------------|
| **H.264 CRF 23~26** | FFmpeg 권장. CRF 23 = “시각적으로 무손실”에 가까움. 26 = 약간 압축, 화질 양호. |
| **오디오 128k AAC** | 웹 스트리밍 표준, 음질 유지. |

→ **적용: ffmpeg 설치 시 scripts/optimize-videos.js 또는 배치 실행. 원본 덮어쓰기.**

## 3. 원본 정리

- 압축·최적화된 파일이 **같은 경로·같은 파일명**으로 원본을 **대체**.
- 별도 백업 폴더 미생성 → **파일 수 2배 증가 없음.**
