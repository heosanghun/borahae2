# 원클릭: 이름·생년월일·사진 → 일관된 건축 + 런웨이 영상

## 이미 정리된 설계

`개발설계_내스토리_런웨이_일관성.md`에 **이름·생일·(스타일링/사진)** 만 넣으면  
**같은 건축물 배경 + 나의 모습이 런웨이를 걸어가는 영상**까지 이어지는 플로우가 정의되어 있습니다.

- **일관성**: “나의 건축물” 1장을 한 번만 만들고, 음악·건축·영상 전부에서 **같은 건축물** 사용
- **데이터**: `myStoryResult` 하나에 스토리·가사·음원·건축 이미지·전신 패션·런웨이 영상 URL 저장

---

## 현실적인 난이도와 선택지

| 단계 | 내용 | 난이도 | 비고 |
|------|------|--------|------|
| 1. 이름·생일·사진 한 번에 받기 | 원클릭 입력 폼 (이름, 생일, 사진 1장) | 낮음 | UI만 정리하면 됨 |
| 2. 이름 스토리 + 탄생뮤직 | 이미 구현됨 (가사·Suno 음원) | 완료 | 가사에 이름 스토리만 더 녹이면 됨 |
| 3. “나만의 건축물” 1장 | 탄생뮤직(가사/설명) → 기존 한글 건축 파이프라인 | 중간 | 입력만 “탄생뮤직” 경로 추가 |
| 4. “나의 전신” 이미지 | 사진 + 추천 옷 → 전신 패션 합성 | 중간 | 기존 스타일링 결과를 “나의 전신”으로 저장 |
| 5. **런웨이 영상** | 건축 배경 + 전신 인물 걸어나오는 **실제 영상** | **높음** | 아래 대안 참고 |

**정리**: 1~4까지는 지금 구조로 단계적으로 가능하고,  
**5번 “진짜 런웨이 영상”**이 가장 비용·기술 난이도가 큽니다.

---

## 5번 런웨이 영상 – 세 가지 방향

### A. 풀 AI 영상 (가장 이상적, 비용·시간 최대)

- **방식**: Runway Gen-3 / Veo 등에 “건축 배경 + 전신 인물” 합성 이미지 1장 넣고 image-to-video
- **장점**: 진짜 움직이는 런웨이 영상, 일관된 배경·인물
- **단점**: API 비용, 생성 시간(수십 초~분), 얼굴/인물 일관성은 아직 불안정할 수 있음

→ **목표는 유지하되, Phase 4 마지막에 도입**하는 걸 권장 (설계서 7. 구현 우선순위 참고).

---

### B. **추천: “움직이는 한 장” (가장 효율적·현실적)**

영상 대신 **한 장의 hero 이미지**를 만들고, 여기에 **짧은 모션**만 넣어 “영상처럼” 보이게 하는 방식입니다.

1. **한 장 제작**
   - “나만의 건축물” 1장(배경) + “나의 전신 패션”(인물)을 **한 이미지로 합성**  
     → 런웨이 포즈로 건축물 앞에 서 있는 최종 이미지
2. **모션 처리**
   - **옵션 1**: 이 이미지에 **짧은 키프레임 애니메이션** (카메라 살짝 줌인/팬, 또는 인물만 미세 움직임) → 5~10초 루프 영상처럼 재생
   - **옵션 2**: CSS/Canvas로 **parallax 스크롤** 또는 **가벼운 모션**만 적용해 “물 흐르듯” 느낌
3. **구현**
   - 배경+인물 합성: 기존 이미지 생성/합성 파이프라인 또는 간단한 오버레이
   - 모션: 프론트에서 `video`(GIF/WebM) 재생 또는 CSS animation으로 처리

**장점**:  
- 원클릭(이름·생일·사진) → “일관된 건축 + 나”가 **한 화면에 확실히** 드러남  
- AI 영상 API 없이도 **일관성**과 **런웨이 느낌**을 줄 수 있음  
- 개발·비용 부담이 적고, 나중에 A 방식으로 교체하기도 쉬움  

→ **“원클릭 + 일관성”을 가장 빠르게 보여주려면 이 방식이 가장 현실적**입니다.

---

### C. 템플릿 런웨이 + 얼굴 합성

- **방식**: 미리 만든 “런웨이 걸어나오는” 영상 1~2종을 두고, 사용자 **사진으로 얼굴만 스왑**
- **장점**: 진짜 걸어다니는 영상, 구현 경로가 명확
- **단점**: “나만의 건축물” 배경과는 별개라서 **배경 일관성**은 포기하거나, 나중에 별도 합성 단계 필요

→ “건축물 일관성”보다 “내 얼굴이 런웨이 걷는 영상”을 우선하면 고려할 수 있는 선택지입니다.

---

## 제안: 지금 단계에서 할 일

1. **원클릭 입력**
   - **이름 + 생년월일 + 사진 1장**을 한 화면/한 플로우에서 받는 UI 추가  
   - 이 세 가지만 있으면 이후 단계 전부에 재사용 가능

2. **데이터 일관성**
   - `myStoryResult`(또는 동일 규격) 하나에  
     `userStory`, `lyrics`, `musicUrl`, `architectureImageBase64`, `fullBodyFashionImageBase64`, (선택) `runwayVideoUrl` 저장  
   - “나만의 건축물”은 **한 번만** 만들고, 이 객체에 넣어서 계속 재사용

3. **우선 완성할 결과물 (B안 기준)**
   - 1~4단계: 이름 스토리 → 탄생뮤직 → **나만의 건축물 1장** → **나의 전신 패션(사진 기반)**  
   - 5단계: **건축 + 전신 인물 합성 1장** → 여기에 **짧은 모션(줌/팬/루프)** 적용해 “런웨이 같은 한 편”으로 재생  
   - 이후 여유가 생기면 Runway/Veo로 **A안(풀 AI 영상)** 추가

이렇게 하면 **“이름·생년월일·사진만 넣으면, 물 흐르듯 원클릭으로, 일관된 건축물 배경에 나의 모습이 런웨이에서 걸어나오는 것처럼 보이는 결과”**를 가장 효율적이고 현실적으로 만들 수 있습니다.
