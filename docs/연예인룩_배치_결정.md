# 연예인 룩 유사 검색: Try-On vs Lookbook 코너 배치 결정

**결정 대상:** "연예인 룩 → 유사 옷 검색" 기능을 **Try-On(Step 6)** 에 넣을지, **Lookbook 코너**에 넣을지.

---

## 현재 구조 요약

| 구역 | 역할 |
|------|------|
| **Lookbook** | "Borahae Signature Lookbook" / "보라해 AI가 제안하는 오늘의 시그니처 코디" — 카드 8장(콘서트, 팬미팅, 데일리, 캐주얼 등) → View Look → 모달(코디 아이템·컬러) → **"이 스타일 시도하기"** → 스타일링 모달 오픈 |
| **Try-On (Step 6)** | 스타일링 모달 안 Step 6. 갤러리/내 의류 업로드 → Try-On 생성·다운로드. (가상 피팅은 2026 상반기 예정) |

---

## 옵션 비교

| 기준 | A. Try-On(Step 6)에 넣기 | B. Lookbook 코너를 "연예인 룩"으로 바꾸고 여기서 유사 검색 |
|------|---------------------------|-------------------------------------------------------------|
| **진입 경로** | 스타일링 시작 → Step 1~5 진행 → Step 6 도달. **최소 5단계** | 홈에서 스크롤 → Lookbook 섹션 → **바로 업로드**. **1클릭** |
| **사용자 의도** | "옷 입어보기" + "유사 옷 찾기"가 한 곳에 섞임 | "연예인 룩 비슷한 거 찾기"가 **단일 목적**로 명확함 |
| **기대감** | Step 6는 "가상 피팅" 기대감이 큼. 유사 검색은 다른 성격 | Lookbook = "룩 보기·찾기" 성격과 **연예인 룩 검색이 잘 맞음** |
| **개발 위치** | 기존 스타일링 모달 내부 수정·확장 | 메인 페이지 Lookbook 섹션만 수정. 모달/Step 로직과 분리 |
| **노출** | 스타일링을 열지 않으면 **노출 안 됨** | 메인 페이지 한 번만 스크롤하면 **항상 노출** |

---

## 권장: **Lookbook 코너에서 연예인 룩 + 유사 검색**

**이유 요약:**
1. **진입이 짧음** — "연예인 룩 비슷한 거 찾기"만 할 때 Step 1~5를 거칠 필요 없음.
2. **의도가 맞음** — Lookbook을 "연예인 룩"으로 바꾸면 "룩 보고 → 비슷한 옷 검색"이 한 플로우로 자연스러움.
3. **노출이 좋음** — 메인 페이지 한 섹션이라 발견률·사용률이 높아지기 쉬움.
4. **Try-On과 역할 분리** — Try-On은 "내 사진 + 옷 → 가상 착용", 연예인 룩은 "사진 올리면 → 유사 검색"으로 명확히 나뉨.

**구현 방향 제안:**
- **Lookbook 섹션 타이틀/설명**을 **"연예인 룩"** 으로 수정 (예: "연예인 룩으로 비슷한 옷 찾기").
- **섹션 상단**에 **"연예인 룩 사진 올리기"** 업로드 블록 + AI 분석 후 **유사 검색 키워드·네이버/무신사/쿠팡 링크** 노출.
- **기존 카드 8장**은 (선택) "참고용 시그니처 룩"으로 유지하거나, 점진적으로 "연예인 룩 샘플"로 교체.

---

## 정리

| 질문 | 답 |
|------|----|
| Try-On으로 하는 게 더 좋을까? | **아니요.** 진입이 길고, "가상 피팅"과 "유사 검색" 성격이 달라 분리하는 편이 좋음. |
| 기존 룩북을 연예인 룩으로 바꾸고 이쪽에서 유사 검색 만드는 게 더 좋을까? | **예.** Lookbook 코너를 "연예인 룩"으로 수정하고, 여기서 유사 검색 기능을 만드는 쪽을 권장함. |

이에 따라 **개발 계획서**의 "대상"을 **Lookbook 코너** 기준으로 수정하는 것을 권장합니다.
