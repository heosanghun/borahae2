# 보라해 연예인 룩 — kpop.fit 분석 및 차별화 가이드

**참고:** [kpop.fit/analyze](https://kpop.fit/analyze) 캡처 기반  
**목표:** 보라해만의 보라 컨셉·철학으로 혁신적 UI/기능 구현

---

## 1. kpop.fit/analyze 기능 요약

| 단계 | 내용 |
|------|------|
| **입력** | 이미지 업로드(드래그/버튼), 최대 5MB, JPG/PNG/WEBP |
| **선택** | 연예인/스타일 힌트(예: NewJeans, BLACKPINK) |
| **실행** | "AI 착장 분석" 버튼, 샘플1~4로 즉시 체험 |
| **결과** | ① 분석 결과 요약(한 문단) ② 스타일 태그(캐주얼, 스트릿, 오버핏 등) ③ 추천 아이템(아우터/상의/하의별 검색어) ④ 검색어 전체 복사 ⑤ 네이버쇼핑/무신사 검색 ⑥ 상세 보기(설명문 + 재검색) ⑦ 추천 상품 리스트(썸네일·가격·바로가기) |
| **디스클레이머** | "링크는 검색 결과로 이동해요. 동일 제품이 아닐 수 있어요." / "브랜드/정확한 모델명은 확실한 경우에만 표시해요." |

## 2. 보라해 차별화 방향 (구체 가이드)

### 2.1 보라 컨셉·철학 반영

| 구분 | kpop.fit | 보라해 제안 |
|------|----------|-------------|
| **컬러** | 다크 + 보라 포인트 | **보라 그라데이션**을 메인으로(퍼플→핑크 `--gradient-primary`), 카드·버튼·강조선에 일관 적용. "I Purple You" 감성 강조. |
| **타이포** | 기능적 텍스트 | **제목은 `--font-display`(Playfair)** 또는 세리프로 한 줄 요약, 본문은 Noto Sans KR로 가독성. "보라해가 읽어주는 요약" 느낌. |
| **공간** | 컴팩트 리스트 | **카드형 레이아웃** + 넉넉한 여백. 결과를 "한 룩 스토리"처럼 요약 → 태그 → 카테고리별 키워드 순으로 시선 유도. |
| **아이콘/장식** | 최소 | **보라빛 작은 포인트**(작은 하트·보라 점)·섹션 구분선에 그라데이션. 과하지 않게 "보라해 시그니처"만 남김. |

### 2.2 기능 차별화

| 기능 | kpop.fit | 보라해 구현 |
|------|----------|-------------|
| **연예인/스타일 힌트** | 텍스트 입력(선택) | ✅ **연예인·스타일 힌트(선택)** 입력창 추가. 프롬프트에 반영해 분석 정확도 향상. |
| **분석 결과 요약** | 한 문단 텍스트 | ✅ AI에게 **한 문장 요약** 요청 → "분석 결과 요약" 영역에 표시. |
| **스타일 태그** | 캐주얼, 스트릿 등 클릭형 | ✅ **스타일 태그 2~4개** 추출 후 칩 형태로 표시(보라 테두리/배경). |
| **카테고리별 검색어** | 아우터/상의/하의 + 검색어 | ✅ 유지. 상의·하의·신발·가방·액세서리 카드형으로 정리. |
| **전체 복사** | 검색어 전체 복사 | ✅ **전체 복사** 버튼 추가. 클립보드 복사 후 토스트 "복사됐어요 💜". |
| **쇼핑 링크** | 네이버/무신사 | ✅ 네이버·무신사·쿠팡 유지. 버튼을 보라 그라데이션 + 호버 시 밝기 올리기. |
| **디스클레이머** | 하단 작은 글씨 | ✅ **두 줄 디스클레이머** 추가. "링크는 검색 결과로 이동해요. 동일 제품이 아닐 수 있어요." / "브랜드·모델명은 확실한 경우에만 표시해요." |

### 2.3 UI/그래픽 체크리스트

- [ ] 연예인 룩 블록 **카드 배경**: `--bg` + 보라 그라데이션 좌측/상단 얇은 라인 또는 soft glow.
- [ ] **드롭존**: 호버 시 테두리만 보라 그라데이션, 아이콘은 "📷" 대신 SVG 카메라·보라 원형 배경.
- [ ] **로딩**: 스피너 색상을 `--primary` 또는 그라데이션.
- [ ] **결과 영역**: 요약 → 스타일 태그 칩 → 검색어 카드 → 쇼핑 버튼 → 디스클레이머 순. 구분선에 `linear-gradient(90deg, transparent, var(--primary), transparent)`.
- [ ] **버튼**: 주 액션(네이버 쇼핑 등)은 `--gradient-primary`, 보조(전체 복사, 다시)는 아웃라인.
- [ ] **다크모드**: 기존 `--bg`, `--text` 변수로 자동 대응.

## 3. 구현 우선순위

1. **연예인/스타일 힌트** 입력 + 프롬프트 반영  
2. **AI 응답 확장**: 요약(summary) + 스타일태그(styleTags) + 기존 카테고리 키워드  
3. **결과 UI**: 요약 문단, 스타일 태그 칩, 전체 복사 버튼, 디스클레이머 2줄  
4. **보라해 비주얼**: 그라데이션 라인, 카드 glow, 버튼/스피너 색상 통일  

이 가이드를 기준으로 현재 연예인 룩 블록을 단계적으로 개선하면 됩니다.
