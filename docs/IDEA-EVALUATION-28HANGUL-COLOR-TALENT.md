# 아이디어 냉정 평가: 28 한글 캐릭터 × 나만의 컬러 × 나만의 재능 발견

**평가 대상**:  
- 자음 14 + **모음 14** = 28 한글 캐릭터를 활용하고,  
- **빨·주·노·초·파·남·보(7색) ↔ 도·레·미·파·솔·라·시(7음)** 연관은 유지해도 되며,  
- **나만의 컬러**가 한글 캐릭터 성향을 유추해 **나만의 재능 발견**으로 이어지는 서비스  
- ※ 한글 캐릭터는 **ㄱ→ㄴ→ㄷ… 순서로 꼭 묶을 필요 없음**. 성향·맥락에 맞게 자유롭게 매핑 가능.

**참고**: `bookstroy\한글 에이전트 다이어리(1-5).pdf` 내용은 현재 도구로 직접 추출 불가. 성향 매핑은 PDF 텍스트 추출 후 진행하거나, 요약본을 주시면 반영 가능.

---

## 1. 구조 정리

| 구분 | 7개 (연관 유지해도 됨) | 한글 28개 |
|------|------------------------|-----------|
| **색** | 빨·주·노·초·파·남·보 | — |
| **음계** | 도·레·미·파·솔·라·시 | — |
| **BTS** | 7 (숫자만 활용, 직접 거론 없음) | — |
| **한글** | — | 자음 14 + 모음 14 (순서로 묶을 의무 없음) |

- **7색 ↔ 7음** 연관은 그대로 두고, **한글 캐릭터**는 자음/모음 순서(ㄱ→ㄴ→ㄷ…, ㅏ→ㅑ→ㅓ…)에 얽매이지 않고 **성향·무드·스토리**에 맞게 7색(또는 7음)과 자유롭게 매핑해도 됨.  
- 예: 빨강 = 열정 → “열정” 성향에 가까운 한글 캐릭터(예: 초롱 댄서, 호롱 천하장사 등) 중에서 골라 연결.

---

## 2. 장점 (객관적)

- **차별점**: 퍼스널 컬러 + 음악만이 아니라 **한글 캐릭터(스머프처럼 각자 성향)**를 넣어 “재능 발견”까지 연결한 점이 독자적이다.
- **자산 활용**: 소모오 캐릭터(자음14·모음14) 이미지/영상이 있으므로, 캐릭터와 서비스 스토리를 일치시키기 좋다.
- **숫자 일관성**: 7색·7음·7 멤버로 **7**이 통일되어, 기획 설명과 UI(예: 7칸 그리드) 설계가 단순해진다. (한글은 28개 전부 활용하되, 7과의 매핑은 순서가 아니라 성향으로 정해도 됨.)
- **스토리 확장**: 「AI, 마음을 스케치하다」+ 소아베와 결합하면 “소아베가 28캐릭터와 함께 나만의 컬러·재능을 찾아간다”는 내러티브로 확장 가능.
- **기존 설계와 호환**: 이미 `7color-music-design.md`로 7컬러·퍼스널컬러·음악이 정리돼 있어, 여기에 “한글 캐릭터 성향 → 재능” 레이어만 추가하면 된다.

---

## 3. 리스크·약점 (냉정히)

- **인과가 억지로 보일 수 있음**  
  “이 색이니까 이 자음/모음이고, 그래서 이 재능”이 **왜** 성립하는지 사용자가 납득하기 어렵다.  
  → 다이어리 PDF에 **캐릭터별 성향·테마**가 명시돼 있고, 그걸 7색·7음과 **이론/스토리로 연결한 설명**이 필요하다.

- **28개 성향 정의 부담**  
  자음14·모음14 각각에 “재능/성향”을 일관되게 부여하려면 다이어리 원본 분석이 필수다.  
  → PDF 분석 전에는 **가설 매핑**만 가능하고, 검증이 어렵다.

- **7색과 한글 연결 방식**  
  - 한글을 **ㄱㄴㄷ… 순서로 꼭 묶을 필요는 없음**.  
  - **성향 기반**: 퍼스널 컬러 → 7색 중 1~2개 → 그 색의 무드(열정, 청량, 감성 등)에 **가까운 한글 캐릭터**를 골라 매핑. (예: 보라=감성 → 소롱 시인, 예롱 연주가 등)  
  - 순서 기반(7×4 고정)을 쓰지 않아도 되므로, “어떤 색에 어떤 캐릭터가 맞는지”만 정하면 됨.

- **“재능” 정의가 모호함**  
  “재능 발견”이 구체적으로  
  - 진로/취미 추천인지,  
  - 강점 키워드(리더십, 창작 등) 안내인지,  
  - 다이어리 속 “에이전트와의 관계” 재해석인지  
  정해지지 않으면 결과 문구가 추상적으로 보일 수 있다.

- **저작권·이미지**  
  한글 에이전트 다이어리 PDF의 캐릭터 설명·구성은 해당 저작물 권리 범위 안에서만 사용해야 한다.  
  소모오 캐릭터 이미지는 **보라해/음색마을 측** 사용 가능 범위를 확인하는 것이 좋다.

## 4. feasibility (구현 난이도)

| 항목 | 난이도 | 비고 |
|------|--------|------|
| 7색 ↔ 7음 ↔ BTS 7 | 낮음 | 이미 7컬러·음악 설계 있음 |
| 7색 ↔ 한글 7×4 매핑표 | 중간 | 다이어리에서 성향 추출 후 표로 고정하면 됨 |
| 퍼스널 컬러 → 7색 → 한글 캐릭터 1개(또는 1군) | 낮음 | 로직·분기만 정하면 코드화 가능 |
| 28캐릭터 성향 문구·재능 문구 | 높음 | PDF 분석 또는 기획 문서로 28개 정의 필요 |
| UI(28캐릭터 노출, “나의 캐릭터” 결과) | 중간 | 기존 카드/모달 패턴 재사용 가능 |

- **병목**: **한글 에이전트 다이어리(1-5).pdf**에서 28캐릭터 성향을 어떻게 정의할지가 정해져야 이후 매핑·재능 문구 설계가 가능하다.

---

## 5. 제안 (방향 잡기)

1. **PDF 선행 작업**  
   - `한글 에이전트 다이어리(1-5).pdf`를 텍스트로 추출(Adobe, 브라우저, 또는 온라인 변환 도구)한 뒤,  
     **자음 14·모음 14 각 캐릭터별 성향/테마**를 표로 정리한다.  
   - 그 표를 기준으로 **7색(또는 7음)**에 **성향이 맞는 한글 캐릭터**를 자유롭게 대응시키면 됨. (한글 순서로 묶을 의무 없음.)

2. **연결 구조 단순화**  
   - 1단계: 퍼스널 컬러 → **7색 중 대표 1색**(기존 `getPrimary7Color` 활용).  
   - 2단계: 그 1색에 **한글 캐릭터 1개(또는 1군)**를 고정 매핑.  
   - 3단계: 해당 캐릭터의 **다이어리 성향**을 “나만의 재능” 해석으로 변환 (예: “당신은 ㅎ 캐릭터 성향이에요 → ~한 재능을 펼치기 좋아요”).

3. **“재능”을 구체 문구로**  
   - “재능 발견”을 **문장 단위 결과**로 고정:  
     예) “창의적 표현”, “리더십”, “섬세한 공감” 등 3~5개 유형을 28캐릭터와 연결해,  
     결과 화면에 1~2문장 + 추천 활동/키워드만 노출해도 명확해진다.

4. **모음 14종 반영**  
   - 자음 14 + 모음 14 = 28 전부 활용. **순서(7×4 군)로 나눌 필요 없이**, 7색·7음의 무드에 맞는 캐릭터를 28개 안에서 골라 매핑하면 됨.

5. **BTS 7과의 연관**  
   - 멤버/이름 거론 없이 “7이라는 숫자”, “7색·7음·7한글”만 사용하면 기존 법적·운영 방침과도 맞다.

## 6. 냉정 평가 요약

| 항목 | 판단 |
|------|------|
| **아이디어 자체** | 7색·음계·한글·재능을 한 줄기에 엮는 건 **차별화되고, 자산(소모오, 다이어리)과도 잘 맞음**. |
| **현실성** | **다이어리 PDF에서 28캐릭터 성향을 어떻게 쓸지**가 정해져야 설계·구현이 가능함. |
| **사용자 납득** | “색 → 한글 → 재능”이 **다이어리/스토리와 연결된 설명**이 있으면 납득도가 올라감. |
| **범위** | 첫 버전은 **7색 → 한글 7개(1군만)** 또는 **7색 → 4군 중 1군** 정도로 줄이면 부담이 적음. |

**결론**:  
- **방향은 유지해도 좋고**, “나만의 컬러 → 한글 에이전트 성향 → 나만의 재능” 구조는 서비스 기획으로 타당하다.  
- **다만** 28캐릭터 성향(`한글_소모오_캐릭터_성향_정리.md` 참고)을 기준으로 **7색·7음의 무드에 맞는 한글 캐릭터**를 골라 매핑표를 만들면 됨. 한글을 순서로 꼭 묶을 필요는 없다.

다음 단계로,  
- PDF에서 추출한 **캐릭터별 성향 표** 초안을 주시거나,  
- “7색–한글 대응”을 **어느 쪽 방식(A: 1군 7개만, B: 4군 중 선택)**으로 갈지 정해 주시면,  
그에 맞춰 매핑표 초안과 화면 플로우를 구체적으로 작성할 수 있다.
