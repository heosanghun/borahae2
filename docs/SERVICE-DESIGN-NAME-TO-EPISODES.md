# [서비스 기획] 이름 → 자·모음 분리 → 에피소드 생성

**참고 자료**: `한글_소모오_캐릭터_성향_정리.md` (한글 에이전트 다이어리 계열 28캐릭터 성향)  
**목적**: 사용자가 **이름**을 입력하면, 이름의 **자음·모음을 하나씩 분리**하고, 각 자·모음에 해당하는 **한글 에이전트**가 등장하는 **에피소드**를 생성하는 서비스 설계.

---

## 1. 컨셉 요약

| 단계 | 설명 |
|------|------|
| **입력** | 사용자 **이름**(한글) |
| **분해** | 이름을 글자 단위로 **초성·중성·종성(자·모음)** 으로 분리 |
| **매핑** | 각 자·모음 → **28 한글 에이전트** 중 1명 (자음 14 + 모음 14) |
| **산출** | 해당 에이전트들이 등장하는 **에피소드(웹툰/스토리)** 시리즈 |

- **한글 에이전트 다이어리**에 등장하는 **자음 14·모음 14 캐릭터**(고롱, 노롱, 아롱, 이롱 등)를 이름에 담긴 “나만의 자·모음”과 연결해, **나만의 에피소드**가 만들어진다는 구조.

---

## 2. 이름 → 자·모음 분리 규칙

### 2.1 한글 음절 분해

- 한 글자(음절)는 **초성(자음) + 중성(모음) + 종성(자음, 있을 때)** 로 구성.
- 예: **민** = ㅁ(초성) + ㅣ(중성) + ㄴ(종성) → **모롱, 이롱, 노롱**  
- 예: **수** = ㅅ(초성) + ㅜ(중성) → **소롱, 우롱**

### 2.2 분리 결과의 순서

- **글자 순서 유지**, 글자 내에서는 **초성 → 중성 → 종성** 순서로 나열.
- 예: **민수** → ㅁ, ㅣ, ㄴ, ㅅ, ㅜ → 에이전트 **모롱 → 이롱 → 노롱 → 소롱 → 우롱**.

## 3. 에피소드 생성 방식 (기획 옵션)

### 3.1 옵션 A: 글자별 에피소드 (1글자 = 1에피소드)

- 이름의 **한 글자마다** 하나의 에피소드.
- 그 글자를 이루는 **초성·중성·종성**에 해당하는 에이전트 2~3명이 함께 등장.
- 예: **민** → 에피소드 1 「민」: 모롱·이롱·노롱이 등장하는 짧은 스토리.  
  **수** → 에피소드 2 「수」: 소롱·우롱이 등장하는 스토리.

**장점**: 이름 글자 수만큼 에피소드가 생겨 구조가 단순하고, “내 이름 한 글자마다 이야기가 있다”는 느낌을 줄 수 있음.

### 3.2 옵션 B: 자·모음 순서대로 1인 1장면 (연속 에피소드)

- 분리된 **자·모음 순서**대로, **한 에이전트당 한 장면(또는 한 컷)**.
- 전체가 **하나의 긴 에피소드**처럼 이어지는 스토리.
- 예: 민수 → [모롱 장면] → [이롱 장면] → [노롱 장면] → [소롱 장면] → [우롱 장면].

**장점**: “이름을 풀어 쓰는 동안 에이전트들이 차례로 등장한다”는 인상, 웹툰 한 편처럼 연속 감상 가능.

### 3.3 옵션 C: 이름 전체 = 하나의 통합 에피소드

- 이름에서 나온 **모든 에이전트**가 **한 편의 스토리**에 등장.
- 각 에이전트의 **성향·역할**을 반영해 대사·행동을 배치 (예: 소롱은 시를 읊고, 우롱은 기록을 남기는 식).

**장점**: “내 이름을 구성하는 친구들이 한자리에 모여 하나의 이야기를 만든다”는 정체성·소속감 강화에 유리.

---

## 4. 에피소드 콘텐츠 설계 (공통)

- **형식**: 웹툰형(이미지 + 말풍선/내레이션) 또는 텍스트 + 캐릭터 일러스트 카드.
- **톤**: 한글 에이전트 다이어리·소모오 캐릭터 톤에 맞춰, **친근하고 따뜻한** 말투.
- **내용**:  
  - 해당 에이전트의 **역할·재능·성향**을 한 줄 정도 드러내고,  
  - “○○(이름)이의 ㅁ/ㅣ/ㄴ…”처럼 **이름과 자·모음**이 자연스럽게 언급되도록 문장 설계.
- **저작권**: 한글 에이전트 다이어리·소모오 캐릭터의 성향·이름·역할은 기존 문서·원저작 범위 내에서만 사용.

---

## 5. 사용자 플로우 (제안)

1. **진입**: PLAY(놀기) 또는 “나만의 에피소드 만들기” 등 CTA.
2. **입력**: 이름 입력란에 **한글 이름** 입력 (2~4글자 권장, 최대 글자 수 정책에 따라 제한).
3. **분해·매핑**:  
   - 입력 이름을 **초성·중성·종성**으로 분리.  
   - 각 자·모음에 대해 **에이전트 매핑표**로 캐릭터 결정.
4. **결과 표시**:  
   - “○○이의 이름에는 [모롱, 이롱, 노롱, 소롱, 우롱]이 숨어 있어요.” 같은 요약 문구.  
   - 선택한 에피소드 방식(A/B/C)에 따라 **에피소드 목록** 또는 **단일 에피소드** 노출.
5. **감상**: 에피소드별로 **웹툰/스토리** 감상 + (선택) 공유·저장·다운로드.

## 7. 기술·구현 참고

- **한글 자·모음 분리**:  
  - 유니코드 한글 음절 식 `0xAC00 + (초성×588 + 중성×28 + 종성)` 역산으로 초성·중성·종성 코드 추출.  
  - JavaScript 등에서 초성/중성/종성 인덱스 테이블로 매핑하면 됨.
- **에이전트 매핑**:  
  - 자음 14·모음 14를 **고정 매핑표**(위 표)로 두고, 분리된 자·모음 코드로 캐릭터 ID/키 조회.
- **에피소드 본문**:  
  - 첫 버전은 **템플릿 문장 + 캐릭터 이름/역할 치환**으로 생성.  
  - 이후 AI 생성 시에는 `한글_소모오_캐릭터_성향_정리.md`의 성향·설명을 프롬프트에 포함해 “다이어리 톤”을 유지하는 것을 권장.
